helm create payment (Helmstructure will be created)
chart.yaml (metadata about the chart)
values.yaml (default configuration values. Use for customization of the templates)
templates/ (contains Kubernetes manifest templates)

helm package payment (Chart is created in the current directory)
helm repo index . (Creates an index.yaml file for the chart repository)
helm lint payment (Checks the chart for possible issues)


# Deploy to dev (default)
helm template my-kube-app ./ --namespace my-kube-app-dev -f values-dev.yaml  --debug
helm install my-kube-app ./my-kube-app --namespace my-kube-app-dev -f values-dev.yaml

# Deploy to prod
helm install my-kube-app ./my-kube-app --namespace my-kube-app-prod -f values-prod.yaml

# Or upgrade
helm upgrade my-kube-app ./my-kube-app --namespace my-kube-app-prod -f values-prod.yaml

# To install a helm chart from a repository
helm install frontend helm/my-kube-app --namespace my-kube-app-dev

# To get yaml files for a chart
helm template my-kube-app ./ --namespace my-kube-app-dev -f values-dev.yaml
